1. PURPOSE

This protocol defines the mandatory testing procedure for integrating and validating the FIN Financial Resilience domain within SPD v13.1.

The purpose is to verify that every FIN scenario:

- Is correctly exposed in the cockpit.
- Passes through the domain integration layer.
- Connects to the authoritative FIN rule engine.
- Loads the correct FIN rules.
- Processes scenario intensity correctly.
- Produces deterministic financial resilience assessments.
- Integrates with Captain AI Lena's authoritative decision architecture.
- Generates an appropriate decision and operational action.
- Updates the Memory Core.
- Produces a complete audit record.
- Passes the SPD v13.1 self-test.
- Passes fault identification.
- Completes corrective action processing when required.
- Passes re-test validation.

A FIN scenario is not considered fully implemented merely because its cockpit button appears or the scenario can be selected.
